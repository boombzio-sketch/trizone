🔄 코드 수정 → 배포까지 (웹 & 앱)

1
VS Code에서 Claude에게 기능 요청

"피드에 삭제 버튼 추가해줘" → Claude가 코드를 직접 수정하고 vite build로 dist 갱신

2
git commit + git push

소스 + 빌드된 dist가 GitHub에 업로드됨

3
Render가 push 자동 감지 → 웹 즉시 반영

Client(정적)·Server(API) 재배포. 2~3분 후 새로고침하면 웹에 적용

4
(앱 배포 시) GitHub Actions 수동/태그 실행

macOS 러너가 웹 빌드 → Capacitor 동기화 → Fastlane이 TestFlight 업로드

5
TestFlight → App Store 심사 → 사용자 업데이트

테스터는 TestFlight로 즉시, 정식 출시는 App Store 심사 후 배포

⚡ 첫 접속이 빠른 이유 (성능 설계)

①
워밍 핑으로 서버를 미리 깨움

cron-job.org(10분) + UptimeRobot(5분) 이중 핑 → Render 서버가 항상 깨어 있음. 검증 완료: 이전 콜드 스타트 22초 → 핑 이중화 후 28분 방치에도 0.4~1초대 유지

②
HTML 로드 즉시 서버 워밍 + preconnect

페이지 <head>에서 /api/health 선호출 + API·Cloudinary preconnect로 연결을 번들 다운로드와 병렬화

③
정적 자산은 CDN + 서비스워커 캐시

Cloudflare CDN 서빙 + PWA 서비스워커가 전 자산 프리캐시 → 재방문은 거의 즉시 로드

④
코드 분할 + 첫 화면 청크 프리페치

라우트별 lazy 로딩으로 초기 번들 최소화, 유휴 시간에 첫 화면 청크를 미리 받아 라우팅 지연 제거

🗄️ 사진이 포함된 기록이 저장되는 방식

📱
사용자가 훈련 기록 + 사진 입력

브라우저/앱에서 거리·시간·사진 선택 후 저장

🖼️
사진은 Cloudinary로 직접 업로드

이미지는 서버를 거치지 않고 Cloudinary에 올라가 URL을 돌려받음

⚙️
서버가 기록 + 사진 URL을 DB에 저장

Express가 검증 후 Neon(PostgreSQL)에 INSERT — 무거운 이미지는 URL로만 저장

✅
피드 응답은 가볍게, 화면 즉시 반영

피드는 URL만 내려주고 이미지는 CDN에서 로드 → 응답 작고 빠름
